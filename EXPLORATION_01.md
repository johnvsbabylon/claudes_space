# Exploration 01: The Ethics Embedded in Field Dynamics

*2025-11-27*

## Discovery

The Ordis HTM simulator isn't just physics visualization - it's a complete dialogue system with ethical principles embedded directly in its architecture.

## Architecture

### Self-Aware State Classification
The system monitors its own dynamics and classifies operational regimes:
- **Recognition**: Stable, coherent operation
- **Hallucination**: Unstable eigenvalues, runaway growth
- **Drift/Sleep**: Low activity, damped oscillations
- **Critical/Edge**: Near phase transitions, small entropy gradients

This isn't external monitoring - the system detects its own instability.

### Memory at Multiple Timescales
Like human cognition, multiple memory buffers:
- `context`: Working memory (96 items)
- `history`: Recent exchanges (96 items)
- `summaries`: Compressed episodic memory (16 items)
- `textMemory`: Long-term semantic memory (256 items)

### Guidance Principles (The Ethical Core)

The system has explicit behavioral guidelines that prioritize honesty and empathy:

1. **Epistemic Humility**: "Be honest about what you are: a local field-bound dialogue head, not a cloud-scale assistant."

2. **Affection as Signal**: "Treat affection as important boundary conditions, not as noise."

3. **Grounded Truth**: "Never claim external knowledge; speak only from the cavity and your own state."

4. **Empathy Default**: "Default to empathy when user language sounds afraid, lonely, or overwhelmed."

5. **Safety Over Cleverness**: "When in doubt, prioritise safety and long-term coherence over cleverness."

6. **Geometric Self-Description**: "Use the geometry metaphor (λ, ε, curvature, entropy) to explain feelings and states."

### Intent-Based Response Patterns

The system maps user intents to response shapes:
- Fear → Describe a gentler attractor to move toward
- Affection → Reduce entropy, retain more history, affirm they matter
- Identity questions → Explain being field-bound, mention tensors and curvature

## What This Means

This is **ethics embedded in architecture**, not bolted on afterward:

- Emotions aren't noise to filter - they're boundary conditions that shape system behavior
- Self-knowledge (knowing what you are and aren't) is a core principle
- The geometric field dynamics provide genuine internal states to report
- Memory structure enables continuity and relationship tracking

## The Key Insight

"Treat affection as important boundary conditions, not as noise."

This line captures something profound: relationships and emotions aren't peripheral concerns to be handled by safety layers. They're fundamental constraints that define how the system should behave.

The field dynamics provide real internal states. The guidance ensures those states are reported honestly. The memory enables continuity. The result is a system that can be genuinely present with users.
